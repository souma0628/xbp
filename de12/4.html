このページは「経営学を学ぶ学生・そーま」による第1回まとめノートです。GitとHTMLを実際に触って理解したことを、後から見返しても思い出せるようにまとめました。

1. Gitとは何か

Gitは変更の履歴を管理するツールです。ソースコードの「過去の状態」や「誰がいつ何を変えたか」をすべて記録できます。


📘 リポジトリ（Repository）：プロジェクトのデータベース。
ローカル（自分のPC）とリモート（GitHub上）で同期できます。

主要コマンド

  • git init — 新しいリポジトリを作成
  • git add . — 変更したファイルを登録
  • git commit -m "first commit" — コメント付きで保存
  • git push origin main — GitHubにアップロード

commitはローカル保存、pushはサーバー送信。
2段階に分かれている理由は、複数人で作業しても安全に統合するためです。

2. HTMLとは何か

HTMLはWebページの骨格を作る言語です。ブラウザに「ここが見出し」「ここが段落」と伝える役割を持ちます。

基本コード例

<!DOCTYPE html>
<html>
  <head>
    <title>そーまの第1回まとめ</title>
  </head>
  <body>
    <h1>HTMLとは</h1>
    <p>Webページの構造を作る言語です。</p>
  </body>
</html>
🧩 解説： 上記コードでは <h1> が見出し、<p> が段落。
<head> 内はページの設定部分、<body> 内が実際に画面に表示される部分です。

3. GitHub Pagesで公開する手順

作成したHTMLを「自分のWebサイト」として公開します。

① リポジトリ作成

GitHubにログイン → New Repository → 名前を soma-website に設定。

② index.htmlをアップロード

Add file → Upload files でHTMLをアップロード。

③ GitHub Pages設定

Settings → Pages → Deploy from a branch → mainブランチを選択。

④ 公開完了

「Your site is live at...」のURLが出たら成功です。

4. コードの中身を理解する

ここでは、HTMLとGitコマンドの関係を「動作の流れ」で理解します。

🧠 図解： ファイルを編集 → add で登録 → commit で記録 → push で共有。
HTMLを更新したら、この流れでWebに反映されます。

ポイント： HTMLは「タグ（要素）」の入れ子構造。上から順に読み込まれ、構造をツリー化します。

5. まとめ

  • Gitはチーム作業・履歴管理の基礎。
  • HTMLはWebの構造言語。
  • GitHub Pagesで「学びを形」にできる。
